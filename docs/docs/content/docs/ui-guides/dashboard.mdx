## Overview

The dashboard serves as your primary landing page in Middleware Manager, providing an at-a-glance view of your system's health and key metrics. It aggregates data from your active data source (either Pangolin or direct Traefik) to give you real-time insights into your middleware infrastructure.

## Key Components

The dashboard surfaces counts for routers, services, and middlewares from the active data source plus quick links to key areas.

<Callout type="info" title="Data source aware">
Counts reflect the currently selected source (Pangolin or Traefik). Switch in Settings to refresh.
</Callout>

### Metrics Display
- **Routers**: Number of HTTP/TCP/UDP routers discovered from your data source
- **Services**: Number of backend services (load balancers, weighted services, etc.)
- **Middlewares**: Number of Traefik middlewares configured in the system
- **API Status**: Connection health to your data source

## Common Use Cases

Use the dashboard to:

- Confirm connectivity (non-zero counts and no API errors).
- Jump to Resources, Middlewares, Services, Plugin Hub, or Security.
- Spot anomalies after deploys (sudden drops in router/service counts).
- Monitor system health during maintenance windows.
- Validate that configuration changes are being picked up by the system.

## Configuration Insights

The dashboard provides valuable insights into your Middleware Manager configuration:

- When counts are zero, it typically indicates a data source connectivity issue
- Sudden drops in counts after a deployment may indicate configuration problems
- Steady increases over time can help you track system growth
- The data source indicator helps you verify you're looking at the correct system

## Troubleshooting

<Callout type="warning" title="Zero counts across all metrics">
If you see zero routers, services, and middlewares:
1. Check your data source selection in Settings → Active Data Source
2. Verify connectivity using Settings → Test Connection
3. Ensure your Pangolin or Traefik instance is running and accessible
4. Check network connectivity between Middleware Manager and your data source
</Callout>

<Callout type="warning" title="API errors displayed">
If you see API error indicators:
1. Review the error message for specific connection details
2. Verify credentials for your data source (if applicable)
3. Check that the API endpoint is correctly configured
4. Look at Middleware Manager logs for detailed error information
</Callout>
